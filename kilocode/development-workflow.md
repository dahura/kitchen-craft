# Development Workflow

## Branching Strategy
- **Main Branch**: `main`
- **Feature Branches**: `<ISSUE_KEY>-<kebab-case-description>`
  - Example: `KIT-123-add-login-page`

## Commit Messages
- Format: `[<ISSUE_KEY>] <description>`
- Example: `[KIT-123] Add login form component`

## Pull Requests
- Create a Draft PR as soon as you push the branch.
- Ensure all CI checks pass before requesting review.
